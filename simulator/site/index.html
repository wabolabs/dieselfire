DieselFire

Open-source ESP32-S3 controller for Chinese diesel heaters

⚠ Alpha — not tested on hardware.
This project is in early development. All UI and protocol development has been done via the PC simulator. Hardware bringup is in progress.
Launch Simulator GitHub

Overview

DieselFire is a modernized open-source controller for generic Chinese diesel heaters, forked from Afterburner. It replaces the original 8-bit AVR design with an ESP32-S3, adding a color touchscreen, WiFi, BLE, MQTT, and environmental sensors on a purpose-built PCB.

Features

  • Blue Wire half-duplex serial protocol at 25000 baud
  • 2.8" 320×240 ILI9341 color TFT with capacitive touch (GT911)
  • WiFi for web interface, MQTT, and OTA firmware updates
  • Bluetooth Low Energy for mobile app control
  • BME280 temperature, humidity, and pressure sensor
  • MQ-7 carbon monoxide detection
  • DS18B20 external temperature probe (optional)
  • 14 programmable timer channels
  • Fuel gauge and hour meter
  • DS3231 RTC with battery backup
  • OTA updates via FOTA and ArduinoOTA

Online Simulator

The browser-based simulator runs the full LVGL touchscreen interface compiled to WebAssembly. It includes a realistic diesel heater emulator with 9 operating states, Blue Wire protocol simulation with CRC validation, and fault injection (bad CRC, partial frames, timeouts) accessible from the BW Debug screen.

Quick Start

git clone https://github.com/wabolabs/DieselFire.git
cd DieselFire
pio run -e dieselfire
pio run -t upload -e dieselfire

Native PC simulator:

cd simulator
cmake -B build && cmake --build build -j$(nproc)
./build/dieselfire-sim

Specifications

MCU
ESP32-S3-WROOM-1-N8R8 (8MB flash, 8MB PSRAM)
Display
2.8" 320×240 ILI9341 + GT911 touch
Connectivity
USB-C, WiFi, BLE 5, Blue Wire (UART)
Sensors
BME280, MQ-7, DS18B20 (optional)
RTC
DS3231 with CR2032 backup
Power
12V DC input
PCB
100×100mm 2-layer, KiCad design

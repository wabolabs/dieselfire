# DieselFire — 3D-Printable Enclosure

Parametric two-part enclosure for the 100 × 100 mm `DieselFire` PCB,
generated by [`generate_case.py`](generate_case.py) using CadQuery.

## Files

| File | Purpose |
|------|---------|
| `dieselfire_case_base.stl` / `.step` | Bottom tray — holds the PCB on standoffs |
| `dieselfire_case_lid.stl` / `.step` | Top lid — screws down onto the base |
| `dieselfire_case_assembled.step` | Both parts mated, for preview/fit-check only |

Regenerate (and tweak any parameter) with:

```bash
cd kicad/case
python3 generate_case.py
```

## Key dimensions

| Parameter | Value |
|-----------|-------|
| PCB | 100 × 100 × 1.6 mm |
| Outer envelope | 106 × 106 × 31.6 mm |
| Wall thickness | 2.4 mm |
| Floor / lid thickness | 2.0 mm |
| PCB standoff height | 10 mm (room for B.Cu connectors) |
| Clearance above PCB | 16 mm (display hangs from lid into here) |
| PCB-to-wall clearance | 0.6 mm per side |

## Display (ILI9341 2.8", lid-mounted)

The 2.8" panel mounts to the **inside of the lid**, facing the user; the PCB
sits behind it in the base. Its FPC tail folds down through the rib notch to the
FH12 connector (U11) on the board. The same geometry is mirrored on the board as
a documentation keepout in [`../pipeline.py`](../pipeline.py) (`DISP_*`).

| Feature | Value |
|---------|-------|
| Active-area window | 43.2 × 57.6 mm (portrait) |
| Glass pocket | 50 × 69.2 mm + 0.4 mm/side fit |
| Bezel ledge | front face stays proud of the glass |
| Retaining rib | 3.5 mm tall, 1.6 mm wall, with FPC exit notch on the +Y edge |

The glass drops into the pocket from the cavity side and seats against the bezel
ledge; the PCB behind it (on its standoffs) backs the panel. Add foam/VHB or
print the optional retention nubs if a tighter hold is needed.

## Buttons (side-actuated)

SW1/SW2 are right-angle tactile switches on the front board edge; their plungers
face the **front wall**, where two Ø7 mm access holes (`SIDE_BUTTONS`) let a cap
or finger reach them. This keeps the front face clear for the touchscreen.

## Mounting

- Four **M3 self-tapping screws** secure the PCB to the corner standoffs
  (pilot Ø2.6 mm) at board coordinates (5,5)/(95,5)/(5,95)/(95,95).
- Four **M3 screws** join the lid to the base through corner bosses
  (lid clearance Ø3.4 mm + Ø6 mm counterbore, base pilot Ø2.6 mm).

## Openings (aligned to measured connector positions)

| Wall | Connectors |
|------|-----------|
| Front (Y≈0) | J2 12 V terminal block, J4 DS18B20, J5 MQ-7, J1 USB-C |
| Left (X≈0) | J3 blue-wire to heater |
| Right (X≈100) | H1 / H2 expansion headers (ribbon slots) |
| Lid window | FH12 display-FPC + DS3231 coin cell access |

Floor and lid carry ventilation slots over the ESP32 / regulator area.

## Recommended print settings

| Setting | Value |
|---------|-------|
| Material | PETG (heat tolerant near a diesel heater) or PLA for bench use |
| Layer height | 0.2 mm |
| Walls / perimeters | 3 |
| Top/bottom layers | 4 |
| Infill | 20 % |
| Supports | None — both parts print flat, openings face sideways/up |
| Orientation | Base: floor down. Lid: top face down (window/bosses up). |

## Assumptions & caveats

- Connector cutouts are sized from PCB **courtyards**; the vertical connectors
  sit a few mm inset from the board edge, so the openings are cable-exit
  windows rather than tight panel-mount bezels. Verify against a printed
  first-article and adjust `CONNECTOR_CUTOUTS` in `generate_case.py` as needed.
- The ILI9341 display is assumed to be **panel-mounted externally** and fed by
  the FH12 FPC through the lid window. If you instead mount the display on the
  lid, replace `LID_WINDOW` with a recessed bezel + screw posts.
- All parameters live at the top of `generate_case.py`; nothing is hard-coded
  downstream.
